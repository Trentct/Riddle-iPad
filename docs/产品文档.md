## 核心定义（2026-07-07 定案）：「一叠有灵魂的纸」

**每一款纸住着一个会回信的灵魂。** 用户为了"和纸上的灵魂对话"打开这个 App：翻到日记本，是那个亲密、好奇的家伙；翻到神谕纸，写下心事浮现一句签文。对话（AI 手写回信）是产品的特点与核心；纸 = 人格 + 视觉样式 + 笔迹，三位一体。

产品化方向（已拍板）：**独立 App Store 产品**，与 iKairos 无关。商业模式假设：免费纸引流，卖纸（新人格+新样式）变现——对创作者型 owner，出新纸即出新内容。
市场注意项（调研部分完成）：中国区 2026 年 4 月《人工智能拟人化互动服务管理暂行办法》将持续性人格化情感互动纳入监管，首发市场倾向海外区（详见调研记录）。

## 纸笔交互语言（2026-07-07 定案）

**这个世界里只有纸和笔，笔是唯一法器。** 全部交互归一为三个动词，新功能必须翻译成三者之一，不许发明第四种：

| 动词 | 场景 | 含义 |
|---|---|---|
| **写** | 纸上 | 一切皆墨迹，交给灵魂回应（写字、涂鸦、画圈都是内容） |
| **圈** | 书桌 | 用笔画圈 = 选择（圈住哪本本子就翻开哪本） |
| **折角** | 纸上 | 笔从纸右下角向内拖 = 合上本子回到书桌 |

三条护栏：
1. **手指等价于笔**——交互语言是"画"，但输入源宽容（App Store 审核员可能没有 Pencil；无笔用户不被挡在门外）
2. **墨与命令靠场景切分**——纸上一切是墨，书桌一切笔画是指令；无模式键
3. **每个笔指令必须有墨迹反馈**——圈选判定宽容（包围面积最大者胜），被圈中者以墨迹变化回应

**书桌**：合上本子后看到的视图，桌上摆着几本本子（纸+人格+笔迹的组合体）。书桌同时是未来"卖纸"的货架——商店入口长在世界观里。

## 是什么 / 不是什么

**是**：一个以"AI 手写回信"为核心的对话 App，多款有人格的"纸"构成产品空间。

**不是**：
- 不是 journaling 工具——不做条目管理、回顾、导出
- 不是通用 AI 陪伴聊天——不卖时长、不做語音、不追求日活时长（卖体验与美学，不卖依赖）
- 不是练字/手写工具——AI 回复是特点，工具化方向已明确排除

灵感与核心算法来自 [MaximeRivest/Riddle](https://github.com/MaximeRivest/Riddle)（reMarkable 平台，MIT），本项目是其 iPad 原生移植 + 产品化延伸。

## 功能

一个完整回合的体验流：

1. 启动即全屏一张米白色"纸"，无任何 UI 元素
2. 用 Pencil 书写（压感笔迹，蓝黑墨色）
3. 停笔 2.8 秒 → 字迹按书写顺序逐笔淡去，像被纸吸走
4. 回信以手写体逐笔浮现：英文用连笔字（Dancing Script），中文用手写楷（霞鹜文楷），按回复语言自动切换
5. 墨迹留驻 8 秒后自己淡去（比用户字迹慢 1.5 倍——"它的字更恋恋不舍"）

体验细节：

- **双语**：模型始终用写信人的语言回复，中英文同一套手写合成管线
- **多轮记忆**：会话内记得之前写过什么（保留最近 3 轮），退出 App 即忘
- **落笔优先**：回信期间用户落笔，回信立即让位，在途请求作废
- **错误也在人设内**：断网/请求失败时，纸上浮现手写的"墨迹晕开了，什么也没显现……"

## 技术方案

**栈**：Swift / SwiftUI / PencilKit / Core Animation / XCTest，XcodeGen 生成工程，零第三方依赖。iPadOS 17+，仅 iPad。

**架构**（TurnEngine 是唯一协调者，模块间互不相识）：

```
DiaryView (SwiftUI)              纸面 + 装配
  ├── InkCanvas (PKCanvasView)   收笔迹
  ├── FadeLayer                  字迹淡去动画
  └── QuillLayer                 AI 逐笔书写动画
TurnEngine                       回合状态机 idle→writing→drinking→replying→lingering
Oracle                           LLM 流式客户端（OpenAI 兼容）
Script                           手写合成纯函数（唯一带单测的模块）
```

**手写合成管线**（本项目的技术核心，移植自 Riddle 的 script.rs）：

```
回复文本 → 128px 光栅化（任意 TTF）→ Zhang-Suen 细化成 1px 骨架
        → 骨架追踪成有序折线 → CAShapeLayer strokeEnd 逐笔回放（900pt/s）
```

关键在于：这条管线对任何字体通用，中文不需要真笔顺数据也能获得"一笔一笔写出来"的视觉效果。

**LLM 接入**：Moonshot `moonshot-v1-8k-vision-preview`（该账号层级无 kimi-latest 权限）。整页手写渲染成 PNG 直接发给视觉模型，识别与回复一步完成（无本地 OCR），SSE 流式返回按句切分——首句到达即开写，API 延迟藏在吸墨动画（2.5~4s）里。`base_url + model + api_key` 三项配置化，切换任何 OpenAI 兼容服务零代码改动。Persona prompt 移植自原作（日记本人格、绝不提 AI、字迹不清则说墨水晕开）。

**密钥管理**：key 只存 gitignored 的 `Secrets.xcconfig`，经 Info.plist 变量注入读取，仓库中无任何密钥痕迹。

**并发安全**（开发中审查修出的三个要点，防回归须知）：
- Oracle 请求失败自动回滚本轮 user turn，保持历史成对
- TurnEngine 用回合代号（turnID）+ 任务取消实现抢占：旧回合的异步续体在每个恢复点自检代号，杜绝旧回信写进新回合
- Oracle 标注 @MainActor，历史记录无跨线程访问

## 关键决策与取舍

- **iPad 而非手机**：魔法感一半靠 Pencil 笔迹质感（压感、笔锋、低延迟），手指书写掉档次。iPhone 适配技术上一行配置可开，留作后续
- **纯 demo 定位**：人格养成/journaling 方向讨论过，有意后置——先验证魔法时刻本身值不值得传播
- **移植而非 fork**：原项目是 Rust/C++ 跑 reMarkable Linux，无一行可复用；实际移植的只有 script.rs 三个函数 + persona prompt，README 已按 MIT 致谢
- **多模态直读代替本地 OCR**：一次 API 调用完成识别+回复，还附带能回应涂鸦
- **MVP 裁剪**：设计稿中"吸墨高斯模糊"（iOS CALayer 动画限制）与"纸面纹理贴图"两项裁掉，纯 opacity 渐隐 + 纯色纸面已满足镜头表现
