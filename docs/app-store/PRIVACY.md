## 1. 我们收集什么

### 1.1 你写下的内容（手写页面图像 + 生成的文字）

- 你用手指或 Apple Pencil 在纸面上写下的笔迹，会在你停笔后被渲染成一张 PNG 图片，
  发送给生成回信的服务，用于生成这一轮的回信（`Oracle.ask(pagePNG:)`）。
- 我们**不会**对你的笔迹做文字识别（OCR）后单独存储"你写了什么"——图片只用于当轮
  请求，不落盘保存。

### 1.2 对话历史（不含图片）

- 每个回信角色（归野/沈砚/Ashford）各自维护一份对话历史，**保存在你的设备本地**
  （App 沙盒的 Application Support 目录，不同步到 iCloud，不上传到我们的服务器）。
- 本地历史只保留：AI 的回信文字（真实内容）+ 一个"你写过字"的占位标记（不含手写图片
  本身）。也就是说，本地存储的是"对话发生过"和"AI说了什么"，不是"你写的原始笔迹"。
- 每个角色最多保留最近 20 轮，超出自动丢弃最旧的记录。
- 卸载 App 会清除这份本地历史（沙盒被系统整体删除）。

### 1.3 匿名设备标识

- 首次启动时生成一个随机 UUID，存储在设备本地（`UserDefaults`），用于服务端按设备
  做每日免费额度限流（`X-Device-Id` 请求头）。
- 这个标识**不与你的 Apple ID、姓名、邮箱等身份信息绑定**，我们无法通过它反查出你是谁。
- 重新安装 App 会生成新的设备标识（等同于免费额度重置），这是刻意的设计取舍，不是
  数据丢失事故。

### 1.4 购买信息

- App 内购买（"解锁无限"，一次性买断）完全通过 Apple 的 StoreKit 处理。我们的服务器
  不接触你的支付方式、账单地址等信息——这些始终只经过 Apple。
- 我们的服务器只会记录"这个设备已解锁"这个布尔状态，不涉及具体交易金额、支付方式等。

### 1.5 我们不收集的内容

- 不收集账号信息——本 App **不需要注册/登录**，没有用户账号体系。
- 不使用任何第三方数据分析/广告 SDK（无 Firebase Analytics、无 Facebook SDK、无
  广告追踪）。
- 不访问通讯录、位置、照片库、麦克风等系统权限（App 只使用 Apple Pencil / 触控输入
  与屏幕渲染，不需要这些权限）。

## 2. 我们如何使用这些信息

| 数据 | 用途 |
|---|---|
| 手写页面图像 | 实时发送给生成式 AI 服务（Moonshot），用于生成这一轮回信文字；不做除此之外的用途 |
| 回信文字历史 | 本地保存，用于让 AI 在下一轮对话中"记得"最近几轮说过什么，实现连续对话感 |
| 匿名设备 ID | 按设备计算每日免费回信次数，超出后提示付费解锁 |
| 购买状态 | 判断是否应绕过每日免费额度限制 |

## 3. 第三方服务

### 3.1 Moonshot（月之暗面）—— AI 回信生成

- 你的手写页面图像与近期对话历史（发给模型的文字部分）会被发送给 Moonshot 的
  视觉大模型 API，用于生成回信文字。
- 是否经由我们自建的后端代理转发（`riddle-backend`，部署于 Railway）取决于 App 的
  `USE_BACKEND` 配置：
  - 若 `USE_BACKEND=NO`（当前默认/开发路径）：App 直接连接 Moonshot API。
  - 若 `USE_BACKEND=YES`（生产路径）：请求先到我们的后端代理，代理透传给 Moonshot，
    再把流式回复原样透传回 App。代理**不落盘保存**你的页面图像或对话内容——它只用
    Redis 记录"这个设备今天用了几次"和"这个设备是否已付费解锁"两个计数器/状态位，
    不记录消息内容本身（见 `riddle-backend/README.md` "What it does"）。
- Moonshot 作为独立的第三方 AI 服务提供商，对通过其 API 处理的数据有自己的隐私与
  留存政策；本 App 无法控制 Moonshot 一侧的数据留存时长。**[Trent 待定：如需更明确的
  披露，建议在此处附上 Moonshot 官方隐私政策链接，或注明"按 Moonshot 服务条款处理"]**。

### 3.2 Apple —— 购买与设备

- App 内购买完全通过 Apple StoreKit 处理，支付信息由 Apple 管理，遵循 Apple 自己的
  隐私政策。
- 应用本身通过 App Store 分发，不涉及额外的 Apple 服务（无 CloudKit 同步、无
  Sign in with Apple）。

### 3.3 没有其他第三方

- 不使用第三方分析、崩溃上报、广告 SDK。

## 4. 数据保留

- **手写页面图像**：不做任何形式的服务端持久化存储——无论走直连 Moonshot 还是走
  我们的后端代理，图像只在生成这一轮回信时被使用，用完即弃。
- **对话历史**：只存在你的设备本地，最多 20 轮/角色，你可以随时删除 App 来清除全部
  历史。我们没有能力远程访问或删除你设备本地的历史（因为它从不上传）。
- **设备使用计数**（每日免费额度、是否付费）：保存在我们后端的 Redis 中，按设备 ID
  做每日重置的计数器，不含对话内容。

## 5. 儿童与年龄

- 本 App 不专门面向儿童设计，也不主动收集面向儿童的个人信息。
- 由于回信内容由生成式 AI 模型产出，其内容边界依赖 Moonshot 一侧的安全策略，
  **不适合确认给 13 岁以下儿童独立使用**。建议的 App Store 年龄分级为 **17+**
  （AI 生成内容不可完全预测），详见 `REVIEW_NOTES.md`。
- 如果你认为儿童在未经监护的情况下使用了本 App 并产生了不当内容，请通过下方联系方式
  与我们联系。

## 6. 你的权利

- **访问/删除本地数据**：对话历史只存在你的设备上，删除 App 即可清除全部本地历史，
  无需联系我们。
- **匿名设备 ID 重置**：卸载重装 App 会生成新的设备标识，等效于"重新开始"。
- **购买记录**：通过 App 内"恢复购买"或 Apple 的"App Store — 购买记录"管理，均由
  Apple 侧维护，我们无法代为查询或篡改。
- **数据可携权/删除请求**：由于我们的服务器不存储可识别到个人的对话内容，通常没有
  额外数据需要导出或删除；如果你认为并非如此（例如怀疑后端日志意外记录了内容），
  请通过下方邮箱联系我们，我们会核实并在合理时间内回复。
